# Inventario de Datos - Google Play Data Safety

## Datos recopilados

| Dato | Tipo | Uso | Almacenamiento | Compartido |
|------|------|-----|----------------|------------|
| Fotos de productos | Imágenes del dispositivo | Crear catálogos PDF | Local (SQLite + archivos) | No |
| Nombre del negocio | Texto | Personalizar catálogos | Local (SQLite) | No |
| Nombre del propietario | Texto | Contacto en catálogos | Local (SQLite) | No |
| Teléfono | Texto | Contacto/WhatsApp | Local (SQLite) | No |
| Email | Texto | Contacto en catálogos | Local (SQLite) | No |
| Dirección | Texto | Contacto en catálogos | Local (SQLite) | No |
| Sitio web | Texto | Contacto en catálogos | Local (SQLite) | No |
| Logo del negocio | Imagen | Personalizar catálogos | Local (archivos) | No |

## Datos NO recopilados

- No se recopilan datos de ubicación
- No se recopilan datos de uso/app activity
- No se recopilan datos financieros
- No se recopilan contactos
- No se recopilan datos de salud
- No se envían datos a servidores externos
- No hay analytics ni tracking
- No hay publicidad

## Permisos solicitados

| Permiso | Justificación |
|---------|---------------|
| Cámara | Tomar fotos de productos para catálogos |
| Almacenamiento (Android < 13) | Acceder a galería para seleccionar fotos |

## Almacenamiento y seguridad

- Todos los datos se almacenan localmente en el dispositivo
- No se utilizan servidores ni servicios en la nube
- La base de datos SQLite está encriptada por defecto en Android
- Los archivos PDF se generan y almacenan localmente
- No se transfiere ningún dato fuera del dispositivo

## Retención de datos

- Los datos se mantienen mientras el usuario los conserve
- El usuario puede eliminar familias, productos e historial individualmente
- No hay eliminación automática de datos
